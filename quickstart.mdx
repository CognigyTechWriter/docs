# Quickstart Guide for AI Agents

In this guide, you will create a Logistics AI Agent to answer general questions and assist with parcel inquiries.

<Steps>
  <Step title="Set Up Prerequisites">
    Configure the necessary LLMs for language and embeddings, for example, OpenAI GPT-4o and ada-002, to enable your AI Agent.
  </Step>
  <Step title="Create Your AI Agent Persona">
    Go to **Build > AI Agents**, set the name (e.g., Peter Parcel), description, avatar, and speaking style. Optionally, assign knowledge and configure security. Click **Create** to test.
    <Frame>
    <video
      autoPlay
      loop

      playsInline
      className="w-full aspect-video"
      src="https://docscognigyassets.blob.core.windows.net/assets/create-ai-agent.mp4"
    ></video>
     </Frame>
  </Step>
  <Step title="Assign a Job">
    In **Build > Flows**, open the assigned Flow, add an AI Agent Node, and assign the persona. Set job title (e.g., Logistics Support Specialist), description, and instructions.
     <Frame>
          <img src="https://docs.cognigy.com/_assets/ai/overview/ai-agent-job-configuration.png" />
        </Frame>
  </Step>
  <Step title="Enhance with Knowledge">
    Upload documents in **Knowledge AI**, then assign the knowledge to your AI Agent in its configuration to improve company-specific responses.
  </Step>
  <Step title="Enable Tool Actions">
    Add Tool Actions to the AI Agent Node, define IDs and parameters (e.g., track_parcel with tracking_code), and use **Resolve Tool Action to** finalize responses.
  </Step>
  <Step title="Configure Memory">
    Set up long-term memory via Contact Profile and short-term memory in the AI Agent Nodeâ€™s **Memory Handling** section. Optionally use Update Profile Node.
  </Step>
  <Step title="Enable Image Recognition">
    Activate **Process Images** in the AI Agent Node and **Attachment Upload** in Webchat v3 Endpoint settings. Ensure a storage provider like AWS or Azure is set.
  </Step>
  <Step title="Deploy Your AI Agent">
    Go to **Deploy > Endpoints**, add a new endpoint, link it to the Flow, and test using **Open Demo Webchat**.
  </Step>
</Steps>
